# Assets de publicación — Checklist Semana 1

## 1. Identidad y ficha

- [ ] Nombre final de app validado en Play Console
- [ ] Categoría correcta (Música y audio)
- [ ] Correo de soporte configurado
- [ ] URL de política de privacidad pública

## 2. Recursos gráficos

- [ ] Icono de app de alta resolución (512 x 512)
- [ ] Feature graphic (1024 x 500)
- [ ] Mínimo 2 capturas por tipo de dispositivo objetivo
- [ ] Capturas con idioma ES-419 (y EN opcional)

## 3. Contenido de la ficha

- [ ] Short description (≤ 80 caracteres)
- [ ] Descripción completa optimizada (beneficios + funcionalidades)
- [ ] Notas de privacidad/permisos coherentes

## 4. Compliance

- [ ] Formulario Data Safety completado y revisado
- [ ] Clasificación de contenido completada
- [ ] App access (si aplica) completado
- [ ] Verificación de anuncios (si aplica: “No contiene anuncios”)

## 5. Release prep

- [ ] `versionCode` y `versionName` listos para release
- [ ] App bundle `.aab` generado en release
- [ ] Internal testing configurado
- [ ] Plan de staged rollout definido

## Entregables de Semana 1

1. Política de privacidad lista para publicar
2. Borrador de Data Safety validado internamente
3. Copy de ficha (short + full description)
4. Lista de assets pendientes con responsables
